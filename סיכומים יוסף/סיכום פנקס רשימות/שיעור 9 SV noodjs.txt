### 🧑‍🏫 **שיעור 9 – Node.js וסביבת הפיתוח בצד השרת**

---

#### 🔧 סביבת עבודה ששמה כותבים JavaScript (Node.js)

- **Node.js** היא סביבת הרצה שמאפשרת להריץ קוד JavaScript **מחוץ לדפדפן**.
- מאפשרת לבנות אפליקציות בצד השרת (Server-side).
- נכתבת ב־JavaScript, אבל **לא מיועדת לעיצוב או תצוגה** – רק ללוגיקה בצד השרת.

> דוגמה: אם באתר שלך יש טופס הרשמה – הקוד שבודק את הסיסמה ומכניס את המשתמש למסד נתונים ירוץ ב־Node.js.

---

#### 💻 קוד פתוח (Open Source)

- Node.js היא **תוכנה חינמית ופתוחה** – כל אחד יכול להשתמש, להוריד, ולשפר.
- יש **קהילה ענקית** שתורמת ספריות (Packages) שאפשר להוסיף למערכת.
- משתמשים בה חברות כמו: Netflix, PayPal, LinkedIn.

---

#### 🖼️ ההבדל בין Front-End ל־Back-End

| תחום | Front-End | Back-End |
|------|-----------|-----------|
| מיקום | בצד הלקוח (בדפדפן) | בצד השרת (ב־Node.js לדוגמה) |
| שפות | HTML, CSS, JavaScript | JavaScript (Node.js), Python, PHP, Java ועוד |
| תפקיד | תצוגה, עיצוב, אינטראקציה עם המשתמש | לוגיקה, מסד נתונים, טיפול בטפסים |

---

#### 🔌 מה זה API?

- ראשי תיבות של: **Application Programming Interface**.
- מתווך בין תוכנה אחת לשנייה – מאפשר לתקשורת בין צד לקוח לצידו של השרת.
- ב־Node.js אנחנו כותבים APIs כדי שהדפדפן יוכל לדבר עם השרת.

> לדוגמה: משתמש ממלא טופס → נשלחת בקשה ל־API שמוסיף אותו למסד הנתונים.

---

#### 🌍 Public API (API ציבורי)

- API פתוח שכל אחד יכול להשתמש בו – ללא צורך בהרשאות מיוחדות.
- לדוגמה: API של מזג אוויר, API של תמונות חופשיות, API של חדשות.

> נוכל לשלוח אליו בקשה ולקבל תשובה בפורמט JSON.

---

#### 🚀 Express.js

- ספרייה (Framework) ל־Node.js שעוזרת לבנות API בצורה נוחה.
- מקצרת לנו המון קוד – מאפשרת להגדיר נתיבים, לטפל בטפסים, לנהל תגובות.

```javascript
const express = require('express');
const app = express();

app.get('/', (req, res) => {
  res.send('שלום מהשרת!');
});

app.listen(3000, () => {
  console.log('השרת רץ על פורט 3000');
});
```

---

#### 🧠 Node.js – סיכום

- JavaScript **בצד השרת**
- רץ על מנוע V8 של כרום
- מאפשר לבנות את ה־Backend של אפליקציות
- קוד פתוח, מהיר, תומך באסינכרוניות
- נהדר לבניית APIs

כעת נפתח את ה CLI ונגיע לתיקיה שבה ניצור את הפרויקט בעזרת הפקודות הבאות: 

dir – מציג את כל הקבצים הקיימים במיקום
cd – פותח את התיקייה עם השם הרצוי (דוגמה: cd desktop)
mkdir – יוצר תיקייה עם השם הרצוי (דוגמה: mkdir newFolder)
type nul > - יוצר קובץ בפורמט והשם הרצוי (דוגמה: type nul > newFile.txt)
node 	– מריץ את כל הפקודות JS שכתובות בקובץ (דוגמה: node index.js)
